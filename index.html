Understanding Load vs. Paint Times

Load Time: When the browser finishes downloading the image data and fires the JavaScript load event.

Paint Time: When the browser actually renders the image visibly on screen.

These times can differ because:

  • The browser might batch rendering updates for performance
  • The browser needs to decode image data after loading
  • Other JavaScript might be blocking the main thread
  • Layout and style calculations need to be performed

Note: First-Contentful-Paint is a page-level metric that measures when the browser first renders any content that gives visual feedback that the page is loading. It's a single event for the entire page, not per image.

Low Quality Image Placeholders (LQIP)

When using LQIP, the browser:

  1. Loads a tiny, low-quality image first (fast to download)
  2. Paints this low-quality image to the screen (gives immediate visual feedback)
  3. Loads the high-quality image in the background
  4. Replaces the low-quality image with the high-quality version when ready

This creates a better perceived performance because users see something quickly, even if the final high-quality image takes longer to load.
